Learning Reflection

"I came in thinking I knew how to build APIs. I left knowing how to build APIs that don't embarrass you at 3 a.m."

Async changed how I think

The notification project was my first real encounter with distributed failure modes. A task timing out, a worker crashing mid-run, a broker going down — none of these were hypothetical. I now default to asking "what happens when this fails mid-way?" before I write the happy path.

Testing is design feedback

Writing integration tests for the auth service exposed three bad decisions in my schema within the first hour. I used to write tests as a final step. Now I use them to pressure-test my assumptions before they calcify into migrations.

Logging is kindness to future-you

I spent two hours debugging a permissions bug in staging before adding structured logs with request IDs. I found the cause in ten minutes after that. I now add structured logging before I deploy anything.

Specificity over coverage

HNG rewards engineers who own something completely over engineers who touch everything lightly. Working at that depth — really understanding a token's lifecycle or a queue's failure modes — is what separates "I used Redis" from "I know Redis."
