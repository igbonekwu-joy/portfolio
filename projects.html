Project / 006

Retry Engine

A fault-tolerant HTTP retry service built for fintech and e-commerce scale. Accepts outbound requests, persists them immediately, and processes them in the background with exponential backoff, jitter, and dead-lettering, so transient failures never lose work.

Built the full retry pipeline, a POST endpoint that returns immediately with a pending status while a background worker handles the actual call. Implemented exponential backoff doubling from 1s to 16s with per-attempt jitter, smart retry logic that retries on 5xx and timeouts but treats 4xx as terminal, and a dead-letter mechanism that stops retrying at maxRetries and never revisits failed requests. Persisted every attempt to a relational DB (sqlite) with full history queryable by request ID or status.
